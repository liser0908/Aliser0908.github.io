Linux建站环境配置

2016-06-16 • 标签: centos6.5 apache PHP MySql

apache 基金会组织
产品有：httpd www服务器
tomcat jsp www服务器…

1.首先检查Linux上是否有apache服务

prm -qa |grep httpd  
prm -qi httpd

如果没有，进行安装 

prm install httpd

启动Apache服务

在网络正常的情况下，启动服务器

service httpd start

查看apache 启动服务是否启动

ps -aux | grep httpd
netstat -natp 查看80端口是否开启
curl http://localhost:80 访问apache默认网页
service iptables stop 确保防火墙已经关闭
关闭SELinux(增强的安全保障) setenforce 0
保证页面的有读的权限
默认页面放在 "/var/www/html"

apache的主配置文件
/etc/httpd/conf/httpd.conf
apache全局配置，可以进行服务器优化

ServerToken OS 告知系统客户端的版本和操作系统
ServerType 系统激活服务器的方式
。。。

监听端口 是使用apache监听在8000端口

Listen 192.170.2.1：8000

监听在特定的ip，可以使得apache只监听外的ip请求

BindAddress 192.168.0.1

主机配置

让其他人可以访问默认网页

chown -R apache:apache /var/www/http 把管理员的权限给它
chomad 400 /var/www/http 只读权限

DocumentRoot 设定服务器的共享路径

apache常见模块

mod_defalte:压缩发送给客户端的内容
mod_rewrite:提供一个基础规则的（使用正则表达式分析器的）实时转向URL请求的引擎。

PHP

apache本不能处理PHP网页
工作原理：用户将请求发送到apache－>判断网页类型－>交付给PHP->php解释器处理（php-fpm【一个服务器程序】）
安装PHP

检查是否安装了PHP
prm -qa |grep php
安装PHP
yum install php
cat /etc/resolv.conf 网络DNS配置    
php -v 查看php版本

安装php-fpm

yum install php-fpm
natstat -natp 检查是否安装

安装Mysql

yum install mysql mysql-server

远程登陆Mysql配置

在mysql的用户表设置用户的host地址
update user set host="" where host="";
刷新权限设置
flush privileges;
利用函数对密码加密
update user set password=Password("123456") ;
本机![navicat设置](http://i.imgur.com/VhJ1jwC.png)

使用Xftp上传phpcms安装网站文件到centos下的某个目录 /home/
再apache的配置文件/etc/httpd/conf/httpd.conf中进行重新配置，将默认文件目录改为 /home/
出现权限问题，更改目录和文件的权限（权限chmod -R 777 文件夹/文件）（）（所属chown -R apache:apache 文件夹/文件），以及关闭防火墙 setenforce 0

每次配置和应该重启一次apache

使用yum search php-（扩展模块名） 来得到扩展模块的链接名
用yum install （链接名） 进行安装
在数据库配置时，localhost可能会出错，可以更改为127.0.0.1
